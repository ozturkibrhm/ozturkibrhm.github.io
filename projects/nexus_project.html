Nexus: Real-Time Data Streaming on Kubernetes

  • Tech Stack: Python, JavaScript, HTML/CSS, Apache Kafka, Zookeeper, PostgreSQL, Grafana, Docker, Kubernetes, YAMLr
  • Github URL: Project Link

This was a group project focused on building a real-time data streaming and monitoring platform on top of Kubernetes. The main goal was to dynamically collect data from WCenter systems based on user-provided IP/port/credentials, stream it in real time, and make it available for storage, analysis, and visualization.

The system architecture consisted of multiple microservices:

  • input-service: Provided a web UI where users entered WCenter connection details, stored as JSON configuration.
  • producer-service (Python): Read configs, connected to WCenter, and published raw data into Kafka topics.
  • Apache Kafka & Zookeeper: Managed message queues to ensure scalable and durable data pipelines.
  • consumer-service (Python): Consumed Kafka messages, processed them, and persisted results into PostgreSQL.
  • Grafana Dashboards: Connected to PostgreSQL for real-time visualization and monitoring of ingested data.

The platform was fully containerized and orchestrated with Kubernetes. Each microservice ran as a Deployment, while Kafka, Zookeeper, and PostgreSQL were deployed as StatefulSets with persistent storage volumes. Sensitive configurations (e.g., database and WCenter credentials) were managed using ConfigMaps and Secrets. Helm charts and YAML manifests automated the deployment process.

The project also integrated CI/CD pipelines (Jenkins/GitLab CI) to automate Docker image builds and Kubernetes deployments. Planned enhancements included Grafana provisioning for automatic creation of dashboards as soon as new data became available.
